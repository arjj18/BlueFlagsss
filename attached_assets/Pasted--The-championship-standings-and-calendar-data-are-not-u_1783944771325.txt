"The championship standings and calendar data are not updating on the live site even after the code has been changed. This is a localStorage caching problem — users who have visited before have old data saved in their browser that is overriding the new correct data. Please fix this permanently.
The core problem
When a user visits the site for the first time the app saves standings data to localStorage. Every subsequent visit loads from localStorage instead of the fresh data in the code. This means any time we update the standings or calendar in the code users never see the changes unless they manually clear their browser storage.
The fix — implement a data version system
Add a version number to all localStorage data. Every time we update the standings or calendar data we bump the version number. When the app loads it checks whether the saved version matches the current version. If they do not match it wipes the old data and loads fresh data from the code.
Add this at the very top of your main app file or wherever the app initialises:
javascript// Bump this number every time you update standings, calendar or any cached data
// Current version reflects the correct 2026 calendar and standings
const APP_DATA_VERSION = '2026-07-13-v1'
const VERSION_KEY = 'pitlane_data_version'

function checkAndClearStaleCache() {
  const savedVersion = localStorage.getItem(VERSION_KEY)
  
  if (savedVersion !== APP_DATA_VERSION) {
    console.log(`Data version mismatch: saved=${savedVersion} current=${APP_DATA_VERSION}`)
    console.log('Clearing stale cached data and loading fresh data...')
    
    // Clear all app-related localStorage keys
    const keysToRemove = [
      'f1_driver_standings',
      'f1_constructor_standings', 
      'f1_standings_meta',
      'f1_calendar',
      'pitlane_standings',
      'pitlane_calendar',
      'generalQuizCompleted',
      'generalQuizStreak',
      'generalQuizLastScore',
    ]
    
    keysToRemove.forEach(key => localStorage.removeItem(key))
    
    // Save the new version so this only runs once per version
    localStorage.setItem(VERSION_KEY, APP_DATA_VERSION)
    
    console.log('Cache cleared. Fresh data will now load.')
  }
}

// Call this immediately when the app loads — before anything else renders
checkAndClearStaleCache()
Important — call this function before anything else
Make sure checkAndClearStaleCache is the very first thing that runs when the app initialises. It must run before any component tries to read from localStorage. If components read localStorage before this runs they will still get the old stale data.
In a React app call it before the root render:
javascript// At the very top of main.tsx or index.tsx before ReactDOM.createRoot
checkAndClearStaleCache()

ReactDOM.createRoot(document.getElementById('root')).render(
  <App />
)
In a vanilla JS app call it at the very top of the script before any other code runs:
javascript// First line of your main script
checkAndClearStaleCache()
